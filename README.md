## Portfolio

Each project targets a different layer of the DevOps lifecycle. Together they form a complete platform.

**[Cerebro](https://github.com/GHFury/Cerebro)** — Platform orchestration. A single GitHub Actions pipeline that provisions infrastructure, deploys an application, verifies health, runs chaos tests, and validates self-healing recovery across all three projects below.

**[Deadpool](https://github.com/GHFury/Deadpool)** — Application and CI/CD. A Flask app with a GitHub Actions pipeline (lint, test, build), Docker containerization, Datadog APM instrumentation, and a chaos engineering API for controlled failure injection.

**[Grogu](https://github.com/GHFury/Grogu)** — Infrastructure as Code. Reusable Terraform modules for Kubernetes (namespace, deployment, service) with a pipeline that runs plan on PRs and apply on merge to main using ephemeral kind clusters.

**[Wolverine](https://github.com/GHFury/Wolverine)** — Configuration management. Ansible roles for server provisioning including security hardening, SSH lockdown, Docker installation, and Datadog agent setup, tested against Docker containers simulating real servers.
